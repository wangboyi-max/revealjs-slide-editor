# tech-debt-todo.md 格式

## 表格格式

```markdown
## 🔴 严重问题

| # | 问题 | 文档描述 | 实际情况 | 发现日期 | 状态 |
|---|------|---------|---------|----------|------|
| TD-001 | **标题** | 文档怎么说的 | 代码实际怎样 | 2026/04/25 | [ ] 待处理 |
```

## Gap 处理规则

1. **新 Gap** → 追加到对应严重级别表格
2. **已存在 Gap** → 检查状态是否变化（可能已修复）
3. **修复完成** → 标记 `[x] 已处理`，移到"已完成"表

## TD 编号规则

- 格式：`TD-XXX`（自增）
- 新 Gap 使用下一个可用编号
- 已存在 Gap 保持原编号
