# AI 对比流程

Agent 会收到以下信息：

## 文档部分

- **docs/index.md** - L0 索引层，知道有哪些文档
- **docs/requirements.md** - L1 摘要，按"覆盖范围/不覆盖范围"分段
- **docs/architecture.md** - 技术架构，包含模块划分、数据流、IPC 等

## 代码部分

根据文档提到的模块名，用 Grep 搜索相关代码文件，读取关键文件内容（组件、store、hooks）。

## Agent 任务

```
你是一个代码审查助手。对比以下文档声明和代码实现，检查是否存在 Gap。

## 文档声明
[从 docs/*.md 读取的内容]

## 代码实现
[根据文档语义搜索到的相关代码]

## 任务
1. 逐项检查：文档描述的每个功能，代码是否实现？
2. 如果实现：实现方式是否符合文档描述？
3. 如果不符：具体差异是什么？
4. 判断严重程度：
   - 🔴 严重：功能完全缺失 或 核心架构不符
   - 🟡 中等：功能部分实现 或 实现与文档描述不符
   - 🟢 轻微：代码质量问题

## 输出格式
按 tech-debt-todo.md 格式输出表格：

### 🔴 严重问题
| # | 问题 | 文档描述 | 实际情况 | 状态 |
|---|------|---------|---------|------|

### 🟡 中等问题
| # | 问题 | 文档描述 | 实际情况 | 状态 |
|---|------|---------|---------|------|
```
